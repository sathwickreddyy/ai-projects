# Intelligent Arch Viewer v2 — Iteration 1 Design Spec

## Problem

v1 tried to build everything at once (custom SVG canvas, ELK auto-layout, palette, review modal, skill adaptation, auth gate) across 65 commits. The result: invisible shapes, broken viewport, tangled edges, unusable canvas. The rendering quality gap between v1 and the reference HTML architecture diagrams (generated by the `arch-diagram-style-full` skill) is massive.

## Goal

Rebuild the viewer as an Excalidraw-like interactive canvas that renders architecture diagrams at reference quality. Start with a focused iteration 1: generate, render, drag.

## What Iteration 1 Delivers

1. Run a new skill on any codebase → generates architecture JSON
2. MCP pushes to backend → stores in PostgreSQL
3. Open URL → React Flow canvas renders with reference-quality custom nodes
4. Drag nodes to reposition → saves positions back via PATCH
5. Pan, zoom, select, edge routing — all built-in from React Flow

## What Iteration 1 Does NOT Include

- Skill adaptation/learning (iteration 2)
- Right panel with callouts, tables, ADRs (iteration 2+)
- Palette drag-to-add new components (iteration 2+)
- AI review features (iteration 3+)
- Auth gate (not needed for local dev)

## Architecture

Three components:

```
┌─────────────────┐     MCP (stdio)     ┌─────────────────┐     HTTP      ┌─────────────────┐
│  Claude Code     │ ──────────────────→ │  MCP Server      │ ───────────→ │  Backend         │
│  + new skill     │                     │  (FastMCP)       │              │  (FastAPI)       │
└─────────────────┘                      └─────────────────┘              │  + PostgreSQL    │
                                                                          └────────┬────────┘
                                                                                   │ REST API
                                                                          ┌────────┴────────┐
                                                                          │  Frontend        │
                                                                          │  (React Flow)    │
                                                                          └─────────────────┘
```

### 1. New Skill (`intelligent-arch-creator-v2`)

A Claude Code skill that scans codebases and generates React Flow-compatible architecture JSON.

**Responsibilities:**
- Scan project files in priority order (configs, entry points, routes, models, services)
- Detect tech stack (Kafka, Redis, PostgreSQL, FastAPI, etc.)
- Generate nodes with type, pixel position, and internal details
- Generate edges with labels, styles (solid/dashed/animated)
- Output JSON that maps 1:1 to React Flow's data model

**Not responsible for:** rendering, storage, or adaptation (iteration 2).

### 2. Backend (FastAPI + PostgreSQL)

Stores and serves architecture session data.

**Tech:** FastAPI, Python 3.12, asyncpg, PostgreSQL 16.

**Schema:**

```sql
CREATE TABLE sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL DEFAULT 'Untitled',
  data JSONB NOT NULL,
  detected_stack TEXT[] DEFAULT '{}',
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

The `data` JSONB column holds the full React Flow state: `{ nodes: [...], edges: [...], viewport: {...} }`. No separate tables for nodes/edges — the entire diagram is one document.

**Endpoints:**

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/health` | Health check |
| POST | `/api/sessions` | Create session (skill pushes here) |
| GET | `/api/sessions/:id` | Get session (frontend loads here) |
| PATCH | `/api/sessions/:id` | Update session (frontend saves drag positions) |
| GET | `/api/sessions` | List sessions |

**Port mapping:** Host 18000 → Container 8000 (same as v1).

### 3. Frontend (React + React Flow)

Interactive canvas with custom node types.

**Tech:** React 18, TypeScript strict, Vite 5, Tailwind CSS 3, React Flow.

**Custom Node Types:**

| Type ID | Component | Visual | Internal Details |
|---------|-----------|--------|-----------------|
| `kafka_broker` | KafkaBrokerNode | Container box with header | Topic rows with partition blocks (P0, P1, P2) |
| `redis_cache` | RedisCacheNode | 3D cylinder (ellipse caps) | Key patterns, role label |
| `postgres_db` | PostgresDBNode | 3D cylinder (blue) | Table names, role badge |
| `api_service` | APIServiceNode | Bordered card with step badge | Endpoint list, sync/async indicator |
| `client_actor` | ClientNode | Card with icon | Type label, description |
| `external_service` | ExternalNode | Rounded card (muted) | Provider, description |
| `container_box` | ContainerNode | Container with header | Nested content rendered as HTML within the node (not separate React Flow nodes) |

Each node type is a React component receiving `data` props from the skill's JSON output. Rendering uses HTML/CSS (not raw SVG) — React Flow handles the canvas mechanics.

**Color Palette (per category):**

| Category | Border | Fill (15% opacity) | Text |
|----------|--------|---------------------|------|
| messaging | `#f87171` | `rgba(248,113,113,0.15)` | `#fca5a5` |
| database | `#60a5fa` | `rgba(96,165,250,0.15)` | `#93c5fd` |
| cache | `#f87171` | `rgba(248,113,113,0.15)` | `#fca5a5` |
| api | `#34d399` | `rgba(52,211,153,0.15)` | `#6ee7b7` |
| infrastructure | `#38bdf8` | `rgba(56,189,248,0.15)` | `#7dd3fc` |
| client | `#a78bfa` | `rgba(167,139,250,0.15)` | `#c4b5fd` |
| external | `#9ca3af` | `rgba(156,163,175,0.15)` | `#d1d5db` |

**Edge Styles:**

| Style | Stroke | Dash | Use |
|-------|--------|------|-----|
| solid | category color, 2px | none | Sync calls (HTTP, SQL) |
| dashed | category color, 1.5px | `5 3` | Async flows (Kafka publish/consume) |
| animated | category color, 2px | animated dash | Active data flow |

**Canvas features (from React Flow):**
- Pan (scroll/drag background)
- Zoom (scroll wheel / pinch)
- Node drag (click + drag any node)
- Edge labels at midpoint
- Minimap (bottom-right corner)
- Fit-to-view on load (`fitView` prop)

**Routing:**
- `/session/:id` — loads and renders a session
- `/` — redirects to most recent session or shows empty state

**Port:** Host 13000 (same as v1).

### 4. MCP Server

Thin pass-through between Claude Code and the backend. stdio transport, launched by Claude Code.

**Tools:**

| Tool | Action |
|------|--------|
| `check_app_health` | GET /health |
| `push_architecture` | POST /api/sessions |
| `list_sessions` | GET /api/sessions |

Three tools only. No skill management tools (iteration 2).

## Testing Strategy

- **Backend:** pytest with httpx async client against test PostgreSQL
- **Frontend:** TypeScript strict (`tsc --noEmit`), manual visual verification
- **Integration:** `docker compose up --build` → push test architecture via MCP → verify render

## Success Criteria

Iteration 1 is done when:
1. `docker compose up` starts all three services cleanly
2. Running the skill on a test codebase generates architecture JSON
3. MCP pushes it to the backend without errors
4. Opening the session URL shows a React Flow canvas with properly rendered nodes matching reference visual quality
5. Dragging a node saves the new position and persists on reload
